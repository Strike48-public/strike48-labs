# Configuration

StrikeKit is configured primarily through environment variables. This guide covers all available configuration options.

## Environment Variables

### Core Settings

#### `SK_PORT`
- **Description**: Local liveview server port
- **Default**: `3030`
- **Example**: `SK_PORT=8080`

```bash
SK_PORT=8080 cargo run --bin sk-connector --features connector --no-default-features
```

### Prospector Studio Integration

#### `MATRIX_HOST`
- **Description**: Prospector Studio gateway endpoint hostname
- **Required**: Yes (for connector mode)
- **Example**: `MATRIX_HOST=connectors-studio.strike48.test`

#### `MATRIX_API_URL`
- **Description**: Prospector Studio API base URL
- **Required**: Yes (for connector mode)
- **Example**: `MATRIX_API_URL=https://studio.strike48.test`

#### `MATRIX_TENANT_ID`
- **Description**: Prospector Studio tenant identifier
- **Required**: Yes (for connector mode)
- **Example**: `MATRIX_TENANT_ID=non-prod`

#### `INSTANCE_ID`
- **Description**: Unique connector instance identifier
- **Required**: Yes (for connector mode)
- **Example**: `INSTANCE_ID=strikekit-connector-1`

### Security Settings

#### `STRIKE48_ACCEPT_INVALID_CERTS`
- **Description**: Accept self-signed SSL certificates
- **Default**: `false`
- **Use Case**: Development environments with self-signed certs
- **Example**: `STRIKE48_ACCEPT_INVALID_CERTS=true`

**Warning**: Only enable this in development environments. Never use in production.

### Logging

#### `RUST_LOG`
- **Description**: Rust logging level
- **Default**: `info`
- **Options**: `trace`, `debug`, `info`, `warn`, `error`
- **Example**: `RUST_LOG=debug`

```bash
# More verbose logging for debugging
RUST_LOG=debug cargo run --bin sk-connector --features connector --no-default-features

# Minimal logging
RUST_LOG=warn cargo run --bin sk-connector --features connector --no-default-features
```

You can also configure per-module logging:

```bash
# Debug only for sk-c2 crate
RUST_LOG=info,sk_c2=debug cargo run --bin sk-connector --features connector --no-default-features
```

## Configuration Examples

### Local Development

```bash
export SK_PORT=3030
export STRIKE48_ACCEPT_INVALID_CERTS=true
export RUST_LOG=info
export INSTANCE_ID=strikekit-dev-1
export MATRIX_HOST=connectors-studio.strike48.test
export MATRIX_API_URL=https://studio.strike48.test
export MATRIX_TENANT_ID=dev

cargo run --bin sk-connector --features connector --no-default-features
```

### Production Deployment

```bash
export SK_PORT=3030
export STRIKE48_ACCEPT_INVALID_CERTS=false
export RUST_LOG=info
export INSTANCE_ID=strikekit-prod-1
export MATRIX_HOST=connectors.strike48.com
export MATRIX_API_URL=https://api.strike48.com
export MATRIX_TENANT_ID=production

cargo run --release --bin sk-connector --features connector --no-default-features
```

### Using Environment File

Create a `.env` file in the project root:

```bash
# .env
SK_PORT=3030
STRIKE48_ACCEPT_INVALID_CERTS=true
RUST_LOG=info
INSTANCE_ID=strikekit-connector-1
MATRIX_HOST=connectors-studio.strike48.test
MATRIX_API_URL=https://studio.strike48.test
MATRIX_TENANT_ID=non-prod
```

Then load it before running:

```bash
# Using bash
set -a && source .env && set +a
cargo run --bin sk-connector --features connector --no-default-features
```

**Note**: Never commit `.env` files containing production credentials to version control. Add `.env` to your `.gitignore`.

## Just Configuration

The `justfile` provides convenient command aliases that use sensible defaults. You can override them:

### Custom Port

```bash
just dev PORT=8080
```

### Custom Prospector Studio Host

```bash
just dev HOST=my-studio-server.example.com
```

### Combined Options

```bash
just dev PORT=8080 HOST=my-studio-server.example.com
```

## Database Configuration

### Database Location

StrikeKit stores data in a SQLite database. The location is platform-specific and cannot be changed via environment variables:

- **macOS**: `~/Library/Application Support/StrikeKit/strikekit.db`
- **Linux**: `~/.local/share/StrikeKit/strikekit.db`
- **Windows**: `%APPDATA%\StrikeKit\strikekit.db`

### Database Backup

To backup your engagement data:

```bash
# Linux
cp ~/.local/share/StrikeKit/strikekit.db ~/backups/strikekit-backup-$(date +%Y%m%d).db

# macOS
cp ~/Library/Application\ Support/StrikeKit/strikekit.db ~/backups/strikekit-backup-$(date +%Y%m%d).db
```

### Database Restore

To restore from a backup:

```bash
# Linux
cp ~/backups/strikekit-backup-20260303.db ~/.local/share/StrikeKit/strikekit.db

# macOS
cp ~/backups/strikekit-backup-20260303.db ~/Library/Application\ Support/StrikeKit/strikekit.db
```

## C2 Configuration

C2 listener settings are configured through the UI, not environment variables:

- **Host**: Interface to bind (e.g., `0.0.0.0`, `127.0.0.1`)
- **Port**: Listener port (default: `8443`)
- **Sleep Interval**: Agent beacon interval in seconds (default: `5`)
- **Jitter**: Random beacon variation percentage (default: `10`)

These settings are stored in the database and persist across restarts.

## Network Configuration

### Firewall Rules

If running C2 operations, ensure the listener port is accessible:

```bash
# Linux (iptables)
sudo iptables -A INPUT -p tcp --dport 8443 -j ACCEPT

# Linux (ufw)
sudo ufw allow 8443/tcp

# macOS
# Add rule in System Preferences > Security & Privacy > Firewall
```

### Proxy Configuration

To route traffic through a proxy, set standard proxy environment variables:

```bash
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
export NO_PROXY=localhost,127.0.0.1
```

## Troubleshooting

### Port Already in Use

If you get "Address already in use" errors:

```bash
# Check what's using the port
lsof -i :3030

# Use a different port
SK_PORT=3040 cargo run --bin sk-connector --features connector --no-default-features
```

### Certificate Errors

If you see SSL/TLS certificate errors in development:

```bash
export STRIKE48_ACCEPT_INVALID_CERTS=true
```

**Remember**: Only use this in development environments.

### Connection Issues

If the connector can't reach Prospector Studio:

1. Verify network connectivity:
   ```bash
   ping connectors-studio.strike48.test
   ```

2. Check DNS resolution:
   ```bash
   nslookup connectors-studio.strike48.test
   ```

3. Test HTTPS connectivity:
   ```bash
   curl -v https://studio.strike48.test
   ```

4. Enable debug logging:
   ```bash
   RUST_LOG=debug cargo run --bin sk-connector --features connector --no-default-features
   ```
