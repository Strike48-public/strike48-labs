StrikeKit is a comprehensive red team operations workstation built with Rust and Dioxus 0.7, designed for professional penetration testers and security researchers. It provides a unified platform for managing engagements, coordinating C2 infrastructure, and documenting findings.

![StrikeKit Dashboard](/img/strikekit/screenshots/dashboard-active-engagement.png)

## What is StrikeKit?

StrikeKit is a full-featured platform that combines engagement management, command and control capabilities, and collaborative documentation in a single application. It runs in **connector mode** as a headless liveview server with hot-reloading support, enabling team collaboration through Matrix integration.

## Key Features

### Engagement Management
Track red team engagements through a complete lifecycle workflow:
- **Planning** - Define objectives, scope, and rules of engagement
- **Active** - Execute operations and track progress
- **Paused** - Temporarily suspend operations
- **Complete** - Finalize and document results
- **Archived** - Historical reference

### Target Tracking
Comprehensive asset management:
- Host discovery and enumeration
- Service identification
- Network topology mapping
- Infrastructure visualization

### Credential Storage
Secure credential management:
- Passwords, hashes, and tokens

## Architecture

StrikeKit is built with a modular crate structure:

```
crates/
├── sk-core/      # Core domain types and business logic
├── sk-db/        # SQLite database layer
├── sk-c2/        # C2 listener and protocol
├── sk-agent/     # Cross-platform agent/implant
├── sk-matrix/    # Matrix connector integration
├── sk-executor/  # Task execution engine
├── sk-workflow/  # Workflow orchestration
└── sk-ui/        # Dioxus liveview UI (connector mode)
```

## Technology Stack

- **Language**: Rust 1.85+ (edition 2024)
- **UI Framework**: Dioxus 0.7 (liveview mode)
- **Database**: SQLite
- **Integration**: Matrix connector protocol
- **Build Tool**: Just command runner
