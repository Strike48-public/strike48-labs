KubeStudio is a native desktop Kubernetes client built with Dioxus and Axum. It runs as a StrikeHub connector or as a standalone application, providing a visual interface for browsing clusters, inspecting workloads, and managing deployments.

## Demo

<img src="/kubestudio-demo.gif" alt="KubeStudio cluster dashboard showing nodes, pods, workloads, and resource navigation" style="border-radius: 8px; border: 1px solid var(--sl-color-gray-5);" />

The dashboard shows a cluster overview with node status, running pods, deployment counts, and recent warnings. The sidebar provides quick access to all Kubernetes resource types — Workloads, Configuration, Network, Storage, RBAC, and Cluster resources.

## Features

- **Cluster browsing** — Navigate namespaces, view workloads, inspect pod details
- **Deployment management** — Scale deployments, view rollout history, trigger restarts
- **Pod inspection** — View logs, resource usage, container status, and events
- **Multi-cluster** — Switch between kubeconfig contexts
- **Dual mode** — Runs standalone or embedded in StrikeHub as a connector

## Architecture

KubeStudio is built as a Dioxus liveview application served by an Axum HTTP server:

```
ks-connector binary
├── Axum server (TCP or Unix socket)
│   ├── /liveview        (Dioxus liveview UI)
│   ├── /health          (health check endpoint)
│   ├── /connector/info  (metadata for StrikeHub)
│   ├── /assets/*        (CSS, JS, WASM)
│   └── /ws              (Dioxus liveview WebSocket)
└── Kubernetes API client (via kube-rs)
```

When running inside StrikeHub, KubeStudio communicates over a Unix domain socket. When running standalone, it binds a TCP port.
